Inputs

Creating input is easy as long as you know how and where to use them. An input consist of several elements:

Label

The label tag defines a label for an input element. The label element does not render as anything special for the userlabel element, it toggles the control.

The for attribute of the label tag should be equal to the id attribute of the related element to bind them together.

-type - this could depend on the needed type.It could be an email, number, passsword and etc. This would restrict the user of typing only according to the type.

-placeholder - it's recomended the input has placeholder since it could facilities the user when filling a form, to understand the purpose of the field.

Help text

Help text should be explicitly associated with the form control it relates to using the aria-describedby attribute. This will ensure that assistive technologies –

such as screen readers – will announce this help text when the user focuses or enters the control.

Feedback text

This type of divs are displayed in case of an error or success, when the user has performed some action:

trying to submit a form, writing wrong email in the input field, password is wrong format and etc.

The color of the message could be different depending on the purpose: warning, success,info or danger. It varies by the class name:

"has-error", "has-info" or "has-warning"
